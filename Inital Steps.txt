How the middleware works and what would be the logic for the middleware?
so this middleware is going to check whether the logged in user is authorised and then would check whether it's a food-partner or not.

In the middleware function authFoodPartnerMiddleware there are 3 parameters - req,res,next
1. first it will check whether there's a token or not
2. then will check whether it is correct token or not in a try catch block
3. try - verify the token using jwt.verify whether the token is correct or not
4. If it gets a data in the decoded variable because in the token there would be an id that was stored while logging in and registering for the food-partner.
5. If there's no value in the decoded variable it will throw an error in the catch block.
6. A next() function would be called which is going to pass all the functionalities to the next controller.

Now the middleware would be called in the router before the controller so first of all the request would be checked at the middleware.

router.post("/", middleware, controller)
